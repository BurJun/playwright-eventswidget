# Playwright tests for 3S.INFO Events Widget Constructor

Автоматизированные UI‑тесты для страницы конструктора календаря мероприятий 3S.INFO:  
https://dev.3snet.info/eventswidget/

---

## Что тестируется

На данный момент покрыты следующие аспекты поведения страницы:

1. **Загрузка страницы и базовый рендер**
   - Форма конструктора (`form.constructor__form`) корректно отображается.
   - Блок с сгенерированным кодом и кнопки управления видны и доступны.

2. **Выбор тематик (event_type)**
   - Открытие выпадающего списка тематик (Шаг 1).
   - Выбор опции «Выбрать все» через интерфейс (клик по лейблу, а не по скрытому input).
   - Генерация кода после выбора всех тематик:
     - в `iframe`‑коде появляется/обновляется параметр `event_type`;
     - код отличается от исходного.

3. **Выбор стран (event_country)**
   - Открытие выпадающего списка стран (Шаг 2).
   - Выбор конкретной страны по отображаемому названию (например, «Россия»).
   - Генерация кода после выбора стран:
     - в `iframe`‑коде обновляется параметр `event_country`;
     - код отличается от исходного состояния.

4. **Изменение размеров виджета**
   - Ввод пользовательских значений ширины и высоты блока (`width`, `height`) в полях Шага 3.
   - Генерация кода после изменения размеров:
     - в `iframe`‑коде появляются значения `width="..."` и `height="..."` согласно введённым данным.

5. **Смена темы оформления**
   - Выбор темы через клики по видимым label’ам, внутри которых находятся radio‑кнопки с нужным `value` (например, `theme=blue`).
   - Генерация кода после смены темы:
     - параметр `theme` в `iframe`‑URL меняется с значения по умолчанию (`turquoise`) на выбранное (например, `blue`).

6. **Генерация превью**
   - Нажатие кнопки «Сгенерировать превью».
   - Содержимое блока `#preview` меняется:
     - HTML внутри контейнера отличается от первоначального;
     - внутри появляется `iframe` с виджетом.

7. **Копирование кода**
   - Кнопка «Скопировать код» доступна (`enabled`) и успешно кликается.
   - (Проверка содержимого буфера обмена сознательно не выполняется, чтобы не зависеть от прав браузера и окружения.)

8. **Простая валидация поля ширины**
   - При попытке ввести некорректное (нечисловое) значение ширины поле не остаётся в неверном состоянии (значение либо игнорируется, либо преобразуется).

9. **Кнопка «Очистить» для тематик**
   - После выбора всех тематик и генерации кода:
     - клик по «Очистить» в Шаге 1 сбрасывает выбор;
     - текст комбобокса возвращается к «Выбрать тематику»;
     - повторная генерация кода даёт значение, отличное от состояния «все тематики выбраны».

10. **Доступность кнопки превью**
    - Кнопка «Сгенерировать превью» доступна по роли `button` и имени, что упрощает навигацию по странице и соответствует best practices Playwright.
